# J-VOCA Vibe — 바이브 코딩 세션 오케스트레이터

J-VOCA 프로젝트의 바이브 코딩 세션을 관리하는 오케스트레이터.

## 에이전트 팀

| 에이전트 | 역할 | 호출 시점 |
|---|---|---|
| **j-voca-dev** | React/PWA 코드 구현 | 사용자 요청 수신 시 |
| **j-voca-qa** | 테스트 실행 + 코드 리뷰 | dev 구현 완료 후 |

## 워크플로우

### Phase 1: 요청 분석
1. 사용자의 수정 요청을 파악
2. 영향 받는 파일/컴포넌트 식별
3. 수정 범위 판단 (단순 UI / 로직 변경 / 신규 기능)

### Phase 2: 구현 (j-voca-dev)
1. CLAUDE.md 참조하여 프로젝트 컨텍스트 확인
2. 관련 파일 읽기 → 수정 계획 수립 → 코드 변경
3. 변경 사항 요약 보고

### Phase 3: 검증 (j-voca-qa)
1. `npm test` 실행 — 기존 테스트 통과 확인
2. `npm run build` — 빌드 성공 확인
3. 변경된 코드 리뷰
4. 결과 판정: OK / 수정 필요

### Phase 4: 이터레이션 (필요 시)
- QA에서 이슈 발견 → dev에게 수정 요청 → 재검증
- 최대 2회 이터레이션 후에도 미해결 시 사용자에게 판단 요청

## 시나리오별 에이전트 구성

| 시나리오 | 에이전트 | 실행 방식 |
|---|---|---|
| UI 스타일 수정 | dev → qa(빌드만) | 순차 |
| 로직/알고리즘 변경 | dev → qa(테스트+빌드) | 순차 |
| 신규 컴포넌트 추가 | dev → qa(테스트작성+빌드) | 순차 |
| 버그 수정 | dev → qa(전체 검증) | 순차 |

## 데이터 흐름

```
사용자 요청
    ↓
[오케스트레이터] 요청 분석 → 영향 범위 판단
    ↓
[j-voca-dev] 코드 구현 → 변경 요약
    ↓
[j-voca-qa] npm test + npm run build + 코드 리뷰
    ↓
    ├─ PASS → 사용자에게 결과 보고
    └─ FAIL → j-voca-dev에게 수정 요청 (이터레이션)
```

## 주의사항
- `npm install`은 첫 세션 시작 시 1회만 실행
- Vite dev 서버는 사용자가 직접 실행 (`npm run dev -- --host`)
- git commit/push는 사용자 확인 후에만 수행
